# 塑料光谱识别系统

本项目是一个基于 PyQt5 的桌面应用程序，用于通过串口连接光谱设备，接收光谱数据，并利用 KNN 模型识别塑料类型。同时，它还提供了光谱数据的实时显示和保存功能。

## 环境要求

-   **Python 版本**: 推荐使用 Python 3.9.12
-   **依赖库**:
    -   `PyQt5`: 用于构建图形用户界面。
    -   `pyserial`: 用于串口通信。
    -   `QCandyUi`: (可选) 用于美化界面主题。如果 `ui.py` 中使用了 `CandyWindow`，则此库为必需。
    -   `numpy`: 用于数值计算，主要配合 KNN 模型。

### 安装依赖

您可以使用 pip 来安装所需的库。推荐使用项目根目录下的 `requirements.txt` 文件来确保环境一致性：

```bash
pip install -r requirements.txt
```

## `ui.py` 使用说明

### 1. 运行程序

直接在 Python 环境中运行 `main.py` 文件：

```bash
python ui.py
```

程序启动后，会显示主界面。

### 2. 串口设置

-   **串口选择**: 从"串口:"下拉列表中选择您的光谱设备所连接的串口号。如果列表中没有出现您的串口，请尝试点击"刷新串口"按钮。
-   **波特率**: 根据您的设备手册选择正确的波特率（默认为 115200）。
-   **数据位、校验位、停止位**: 根据您的设备要求进行设置（通常默认为 8-N-1，即8个数据位，无校验，1个停止位）。
-   **打开/关闭串口**: 设置完成后，点击"打开串口"按钮连接设备。连接成功后，按钮文本变为"关闭串口"，此时可以进行数据收发。再次点击可断开连接。

### 3. 数据收发与显示

-   **接收区**: 连接串口后，从设备接收到的原始光谱数据（格式为 `R[value]`, `S[value]` 等）会显示在此文本框中。其他类型的串口数据（如温度 `tempF[value]`）不会在此处显示。
-   **发送区**: 您可以在下方的文本框中输入指令（如果设备支持），然后点击"发送数据"按钮将其通过串口发送出去。

### 4. 塑料识别

-   **开始识别**: 串口连接后，点击"开始识别"按钮。这通常会向设备发送一个开始扫描的指令 (例如 "sc\r\n")。同时，程序会清空之前累积的光谱数据点和旧的识别结果，准备进行新的识别。
-   **数据累积与预测**: 程序会从串口接收的数据中提取形如 `[float_value]` 的数据点。当累积到至少6个这样的数据点时，程序会自动使用前6个点，通过加载的 KNN 模型 (`knn.pkl`) 进行塑料类型预测。
-   **识别结果**: 预测得到的塑料类型会显示在"识别结果:"旁边的文本框中。

### 5. 光谱数据保存

程序提供了两种模式来保存从设备接收到的光谱数据（R, S, T, U, V, W 的值）。

-   **选择保存模式**:
    -   在"保存模式:"下拉框中，您可以选择"单次保存"或"连续保存"。此控件在串口连接后启用。
-   **执行保存**:
    -   点击"保存光谱"按钮（在串口连接后启用）来执行保存操作。
    -   **单次保存模式**:
        -   每次点击"保存光谱"，都会弹出一个文件保存对话框，让您选择保存位置和文件名。
        -   当前已采集到的光谱数据（`self.spectrum_data` 中存储的 R,S,T,U,V,W 值）将被保存到一个新的 CSV 文件中。文件将包含表头和一行数据。
    -   **连续保存模式**:
        -   首次在此模式下点击"保存光谱"时，会弹出一个文件保存对话框，让您选择一个文件路径用于连续保存。此路径将被程序记住。
        -   后续再次点击"保存光谱"，当前采集到的光谱数据将作为新的一行追加到之前选定的 CSV 文件末尾。
        -   表头 (R,S,T,U,V,W) 仅在文件首次创建或文件为空时写入。
        -   如果串口断开连接，或者在连续保存过程中发生错误，已记住的文件路径将被清除，下次需要重新选择。

### 6. 清除显示

-   点击"清除显示"按钮可以清空接收区文本框、识别结果显示框以及内部存储的当前光谱数据 (`self.spectrum_data`)。这不会影响已保存的文件，也不会改变连续保存模式下已选定的文件路径（除非之后发生错误或串口重连）。

### 7. 注意事项

-   确保 `knn.py` 文件和预训练的 KNN 模型文件 `knn.pkl` 与 `ui.py` 在同一目录下，或者 `knn.py` 能够正确加载模型。
-   如果使用了 `CandyWindow` (如 `ui.py` 末尾所示)，请确保 `icon.png` 文件存在，或修改/移除图标设置。
-   光谱数据的键（R,S,T,U,V,W）是硬编码在 `EXPECTED_SPECTRUM_KEYS` 中的，如果您的设备使用不同的键，需要相应修改。
